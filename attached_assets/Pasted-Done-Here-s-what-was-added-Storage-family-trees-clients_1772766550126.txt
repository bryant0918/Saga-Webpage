People route (family_trees/web/api/people/views.py): Three new POST endpoints, all sharing the same internal handler:
Endpoint	Returns
POST /people/tree/husb	Husband ancestor tree JSON
POST /people/tree/wife	Wife ancestor tree JSON
POST /people/tree/kids	Children JSON
Each endpoint expects a JSON body with:
{
"title": "My Tree Name",
"family_search_id": "ABCD-1234"
}

Responses:
* 200 — the stored JSON data
* 400 — missing body or missing required fields
* 404 — the JSON hasn't been stored yet for that title/ID
* 500 — unexpected error


Here's the structure for each JSON object returned:

/people/tree/husb and /people/tree/wife
Both return the same structure — a dictionary keyed by person ID, representing the person and all their ancestors up to the max generation limit.
{
"ABCD-1234": {
"name": ["John Henry", "Smith"],
"birth": "12 Jan 1850",
"death": "5 Mar 1920",
"image": "/path/to/image.jpg",
"parents": ["EFGH-5678", "IJKL-9012"]
},
"EFGH-5678": {
"name": ["William", "Smith"],
"birth": "3 Apr 1820",
"death": "19 Nov 1875",
"image": "/path/to/image.jpg",
"parents": ["MNOP-3456", "QRST-7890"]
},
...
}

Field	Type	Notes
name	[string, string]	First/middle names, then last name as a two-element array
birth	string | null	Formatted date e.g. "12 Jan 1850"
death	string | null	Same format, or empty string if unknown
image	string	File path to the profile image
parents	[string, string]	Array of two person IDs — father first, mother second
/people/tree/kids
A dictionary keyed by child's person ID. Similar to the above but simpler — no death or parents fields, and adds birth_year as a separate integer.
{
"UVWX-1357": {
"name": ["Mary Jane", "Smith"],
"birth": "4 Jun 1878",
"birth_year": 1878,
"image": "/path/to/image.jpg"
},
"YZAB-2468": {
"name": ["Thomas", "Smith"],
"birth": "11 Sep 1881",
"birth_year": 1881,
"image": "/path/to/image.jpg"
}
}

Field	Type	Notes
name	[string, string]	First/middle names, then last name
birth	string | null	Formatted date e.g. "4 Jun 1878"
birth_year	integer | null	The year extracted from the birth date
image	string	File path to the profile image
